AuxiliaryKeyManager

AuxiliaryKeyManager<KA, KO, V> class Null safety

A class used to manage conversions between a main key of type KO and an auxiliary key of type KA.

KA is required to implement operator== and hashCode as if it were a key in a standard Map.

Auxiliary key associations work as follows: +-->AuxiliaryKeyManager.generateAuxiliaryKeyFromMainKeyAndValue--+ | V (Main Key) (Auxiliary Key) ^ V +-AuxiliaryKeyManager.generateMainKeyFromAuxiliaryKeyAndValue----+

Auxiliary keys associate with other auxiliary keys through the associated main key.
